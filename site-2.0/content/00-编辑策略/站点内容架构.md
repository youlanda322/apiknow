# APIKnow 博客内容架构

## 定位

APIKnow 是以 API 工程知识为主的深度博客，不以服务商列表和价格卡片作为首页主体。核心读者是开发者、技术负责人、AI 产品团队和需要进行 API 选型的业务团队。

## 主导航

1. 首页：主线专题、最新文章、系统化系列。
2. API 基础：HTTP、鉴权、签名、分页、错误语义、版本控制。
3. 模型与接口：LLM、图像、语音、Embedding、Rerank、工具调用。
4. 实战开发：SDK、流式、异步任务、回调、重试、缓存、队列。
5. 测评对比：评估方法、场景对比、成本模型、迁移和退出成本。
6. 故障排查：429、5xx、超时、DNS、TLS、尾延迟、配额和事件复盘。
7. 行业观察：协议变化、模型版本、平台政策和 API 生态。

## 首页内容顺序

1. 品牌与编辑承诺。
2. 本期主线专题。
3. 五个热门主题入口。
4. 一篇首发深度文章与编辑部关注。
5. 最新深度文章。
6. 系统化专题系列。
7. 编辑原则与固定披露。

## 内容深度标准

- 入门文章：解决一个基础概念，必须包含例子和常见误区。
- 技术解析：解释机制、边界、失败方式和可观测信号。
- 实战教程：提供可运行步骤、环境、代码、验收和回滚。
- 测评对比：公开方法、数据、样本、限制和商业关系。
- 故障排查：从现象到证据，给出分层定位顺序和复盘清单。
- 行业观察：区分事实和判断，说明变化对开发与业务的实际影响。

## 暂不作为主线的内容

- 仅改写官方公告的短新闻。
- 没有测试条件和证据的“十大推荐”。
- 只比较首页单价的厂商排名。
- 要求用户上传 API Key 的在线检测服务。
- 批量生成、主题相近但没有新增信息的 SEO 页面。
