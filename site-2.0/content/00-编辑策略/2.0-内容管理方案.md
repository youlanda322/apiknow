# APIKnow 2.0 内容管理方案

## A. 静态 2.0（当前已实现）

使用 `editor.html` 添加标签、保存浏览器草稿并下载 Markdown。适合个人维护和验证内容方向，零后台成本，但发布仍需把文件放入内容目录或提交 Git。

## B. Git 内容后台

使用 Decap CMS 或 Pages CMS 管理 Markdown。适合 1-3 人持续写作，可在浏览器中维护标签、草稿和发布时间，同时保留 Git 历史。文章达到 30-50 篇后建议升级。

## C. Headless CMS

使用 Payload 或 Strapi 管理文章、站点、测评记录和证据。适合多人编辑、权限审核和大量结构化数据，但需要服务器、数据库、备份和安全维护。

## 推荐路线

1. 当前先使用静态 2.0，验证栏目、标签和评分规则。
2. 写作频率稳定后接入 Git 内容后台，不改变 Markdown 数据。
3. 中转站数据需要多人维护或商用 API 后，再迁移到 Headless CMS。
